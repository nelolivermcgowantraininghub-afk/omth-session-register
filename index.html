📚 OMTH Session Register — Guide
A reference for everything in the tool — statuses, colours, badges, capacity rules and site priorities.
🏳 Session Statuses
Upcoming Session is scheduled and in the future. No action needed unless experts or trainer are missing.
In Progress Session is happening today and has not yet passed 4:30pm. Updates automatically.
Completed Session date has passed (or passed 4:30pm on the day). Marked automatically. Hidden by default — filter by Completed to view.
Cancelled Session has been cancelled. Hidden from the default view — filter by Cancelled to view.
Cancelled (Ed Centre) Room has been cancelled with the people in charge of bookings at that centre. Hidden by default.
No Capacity No trainer can be assigned — both Jody and Bili are already at their limit for that day. Auto-detected, but can also be set manually. Visible in the status filter.
Unassigned — Capacity Available Session has no trainer yet, but there IS capacity available on that day. Filter for these to see which TBC sessions can still be filled.
TBC Shown on the Main Trainer cell when no trainer has been assigned yet. Shows in the Show Gaps popup.
🎨 Row Highlighting
Green left border Today's session — happening right now.
🔴 Red row Hard conflict — trainer is unavailable (annual leave/day off), same trainer booked twice in a day, more than 2 sessions on one day, or session is after 23 December.
🟠 Amber row Capacity warning — Bili over 3 sessions this week, Jody over 6 sessions this month, or combined Jody + Bili over 5 sessions this week.
Red date highlight Date cannot be parsed — the date format is unrecognised. Edit the session to fix it.
⚠️ Capacity Icons (next to trainer name)
🔴 Hard conflict. Hover for details — e.g. "Jody unavailable: Annual leave" or "Bili has more than 1 session on this day".
🟠 Capacity warning. Hover for details — e.g. "Bili over weekly limit (4/3)" or "Combined weekly limit exceeded (6/5)".
🟡 Soft warning — e.g. Bili scheduled on a non-standard Friday (even week number).
🏢 Site Priority (shown in Coverage Gaps popup)
P1 Mile End — Highest priority. Fill these first. If capacity is tight, Mile End sessions should be protected.
P2 Royal London — Second priority.
P3 Homerton — Lowest priority. If sessions need to be rescheduled or dropped due to capacity, Homerton is flagged first.
👥 Trainer Capacity Rules
Jody
• Max 6 sessions per month
• Max 1 session per day
🚫 Unavailable: 7–13 Sep (annual leave)  |  23–25 Sep (university)  |  4 Dec (day off)
Bili
• Max 3 sessions per week
• Max 1 session per day
• Works every other Friday only (odd ISO week numbers — shown as 🟡 soft warning if on even week)
🚫 Unavailable: 12–23 Oct (annual leave)
Combined (Jody + Bili)
• Max 5 sessions per week between them
• Max 2 sessions per day — one Jody session and one Bili session
🚫 No delivery after 23 December 2026

☁️ Cloud Sync
• All changes sync automatically to Firebase after every edit.
• The tool polls for changes every 15 seconds — if a colleague makes a change, your view updates automatically.
• Sessions also push to the shared Google Sheet (Barts sites only, confirmed trainers).
• If the cloud is unavailable, the tool works offline using the built-in embedded data.
• Use ♻ Restore Built-in Data in the sidebar if sessions ever go missing.
